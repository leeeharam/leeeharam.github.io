스택(Stack)은 후입선출의 개념으로 먼저 넣은 자료가 가장 나중에 나오는 자료구조이다. 순서대로 1, 2, 3을 넣었다면 밑바닥부터 차곡차곡 1, 2, 3이 쌓이는것이다. 여기서 중요한 점은 자료를 꺼냈을때 가장 먼저 넣은 1이 아닌 가장 나중에 넣은 3이 나온다는 것인데, 이와 상반되는 개념으로 큐(Queue)가 있다.

코틀린에서 스택을 만드는 방법은 복잡하지 않으며 List를 선언하는것과 방법이 같다. 첫번째 방법은 간단하게 스택을 선언하는 것이고 두번째 방법은 형식을 같이 넣어주는 것이다. 두 방법 모두 같게 작동된다.

val arr = Stack<Int>()
val arr: Stack<Int> = Stack<Int>()

// 출력: (공백)

스택에서 사용되는 명령어는 가장 기본적으로 push와 pop이 있다. push의 경우에는 자료를 하나하나 차곡차곡 쌓아서 넣는것이고 반대로 pop은 넣었던 자료를 위에서부터 하나씩 하나씩 꺼내는 것이다. 추가적인 명령어로 스택의 크기, 스택이 비어있는지, 스택에 제일 위에는 어떤 자료가 있는지 확인하는 코드도 있는데 각각 size, empty(), peek()를 이용한다. 또한 스택을 완전히 비워버리는 clear()도 있다.

이 경우에는 arr이라는 스택에 1, 2, 3을 넣고 가장 나중에 넣은 자료인 3을 꺼낸 후, 1과 2를 출력하는 코드다.

arr.push(1)
arr.push(2)
arr.push(3)
arr.pop()
println(arr)

// 출력: [1, 2]

1을 넣고 pop을 해주면 당연히 1이 스택에서 사라질텐데, 사라지는 자료인 1을 출력하는 코드이다.

arr.push(1)
println(arr.pop())

// 출력: 1

반대로 이런 경우에는 스택에 1이 들어가면서 1이 출력된다.

println(arr.push(1))

// 출력: 1

출력은 리스트와 똑같은 개념으로 동작하므로 arr[0]을 출력하려고 할시 1이 출력된다.

arr.push(1)
arr.push(2)
println(arr[0])

// 출력: 1

스택이 비어있는지, 비어있지 않은지 체크를 할 수도 있는데 이때는 if문을 이용한다.

if (arr.empty()) {
    println("스택이 비어있습니다.")
}

// 출력: 스택이 비어있습니다.

peek라는 코드를 이용하면 가장 최근에 넣은, 그러니까 가장 위에 있고 pop을 한다면 사라지는 자료형을 출력 할 수가 있다. 단지 arr.peek()라고만 코드를 입력하면 push나 pop처럼 출력의 유무와 관계없이 부가적인 기능을 수행하는 것이 아니므로 출력을 병행해줘야 한다. 이 경우에는 2가 출력이 된다.

arr.push(1)
arr.push(2)
println(arr.peek())

// 출력: 2

스택에 들어간 자료형이 2개이므로 사이즈로는 2가 출력된다.

arr.push(1)
arr.push(2)
println(arr.size)

// 출력: 2

clear라는 코드를 이용하면 스택에 들어있는 모든 내용을 삭제하는데, 이 경우에는 스택에 들어간 자료를 모두 삭제하고 스택을 출력하므로 아무것도 출력되지 않는다.

arr.push(1)
arr.push(2)
arr.clear()
println(arr)

// 출력: []
